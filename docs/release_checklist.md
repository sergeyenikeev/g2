# Release Checklist

## Перед сборкой
- Убедиться, что выбран один из поддерживаемых таргетов: `generic`, `yandex`, `vkplay`, `rustore`.
- Проверить целевой режим (`.env.<platform>`) и что `VITE_PLATFORM` выставлен корректно.
- Для production-артефактов выключить моки (`VITE_USE_PLATFORM_MOCK=0` или удалить переменную).
- Актуализировать `promo/metadata.json` и проверочные скриншоты.
- Убедиться, что нет критичных открытых пунктов в `docs/defects.md`.

## Сборка и артефакты
- Выполнить:
  - `npm run lint`
  - `npm run test`
  - `npm run build:<platform>`
- Проверить, что в `dist/<platform_or_dist_name>` есть:
  - `index.html`
  - JS/CSS/asset-файлы сборки
  - корректные относительные пути (`base: "./"`).

## Смоук после сборки
- Прогнать локальный preview (`npm run preview`) и проверить:
  - загрузку всех экранов;
  - старт Tutorial / Play / Daily;
  - сохранение прогресса после перезагрузки;
  - корректную деградацию при отсутствии SDK/рекламы.
- Для порталов с реальным SDK проверить минимум один midgame и один rewarded сценарий.

## После выката
- Открыть билд в целевой платформе и повторить короткий smoke.
- Проверить, что нет блокирующих ошибок в консоли.
- Подтвердить корректность метаданных: название, описание, теги и скриншоты.

## Релизные заметки
- Зафиксировать изменения в `CHANGELOG.md`.
- Добавить ограничения и известные риски из `docs/defects.md`.
