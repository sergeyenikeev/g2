# OPS рекомендации и развертывание

## Развертывание
- Основной артефакт: содержимое `dist/<platform_or_dist_name>`.
- Для веб-порталов загрузка выполняется как статический набор файлов без backend.
- Для Rustore Android используется WebView-обвязка из `docs/rustore_android.md`.
- Для отката стоит хранить минимум две предыдущие версии ZIP-артефактов.

## Обслуживание и мониторинг
- Логи:
  - игровой лог через `src/utils/logger.ts` (`INFO/WARN/ERROR`, `DEBUG` только в dev);
  - ошибки и предупреждения из консоли анализировать во время smoke-тестов.
- Метрики:
  - crash-free запуск;
  - успешные сценарии Play и Daily;
  - корректные переходы в Results и rewarded flow.
- Бэкапы:
  - хранить релизные ZIP-файлы;
  - версионировать `promo/metadata.json` и ключевые документы.
- Алерты:
  - для CI - fail по `lint/test/build`;
  - для ручных релизов - блокировать публикацию при критичных дефектах в `docs/defects.md`.

## Рекомендации для разработки
- Перед merge и release выполнять:
  - `npm run lint`
  - `npm run test`
  - `npm run build:<target>`
- Не хранить секреты в репозитории; использовать `.env` и секреты CI.
- Не вносить платформенную SDK-логику напрямую в `App.ts`; использовать только `PlatformBridge`.
